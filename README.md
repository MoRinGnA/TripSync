# AI 여행플래너 Project-P (임시)

다중 사용자가 AI를 통해 빠르고 편리하게 여행을 계획할 수 있는 웹 플래너

지금까지 구현한 서비스 테스트 링크: https://project-4gwlyokn9-mo-rin-a.vercel.app/

**로드맵**

1주차 (완료)
  * 개발환경 세팅 (VSC, React, Vite, Tailwind css)
  * GitHub repository 연동 및 초기 코드 업로드
  * 여행 일정 UI 임시 구현

2주차 (완료) 
  * 스타일 요소 index.css 로 분리
  * 탭 분리기능, 수정버튼, 텍스트 클릭 시 수정기능 추가.
  * UI 스타일 변경
  * useEffect를 사용한 저장기능 구현
  * dnd-kit 사용
  * 직관적인 n 일차 블록의 드래그 & 드롭 구현
  * 날짜 밀림 구현

3주차 (완료)
 * 카카오맵 API키 발급
 * App.jsx 파일에 몰려있던 모든 파일을 컴포넌트 단위의 기능별로 분리
 * 컴포넌트 파일에 적용할 CSS 파일 또한 styles 파일 내부로 새로 작성
 * 카카오맵 탭 UI 구현 및 카카오탭 API 연동 성공
 * 구글맵, 카카오맵 변경 UI 구현 및 구글맵 API 연동 성공
 * 지도에서 검색한 장소를 일정에 추가하는 기능 추가

4주차 (완료)
 * Gemini AI API 키 발급
 * AI API 연동
 * AI 일정 자동 생성 탭 추가
 * 화면 분할 탭 구현
 * 맵 검색 로직 개선
 * 일정 클릭 시 지도 탭으로 이동 기능 구현
 * 지도 탭에서 장소 겁색 시, 장소에 맞는 지도 프로그램 사용하는 로직 구현

5주차 (완료)
 * github pages에 페이지 테스트 배포 성공
 * 프론트엔드 직접호출 과정에서 API키 leak로 인한 AI API 사용중지
 * 이를 방어하기 위해서 백엔드 서버를 거치는 과정을 통해서 키의 차단을 억제
 * 해당 서버를 구현하기 위해서 Node.js를 사용하여 구축
 * Pages의 경우 백엔드 서버를 구현해주지는 못하기에, Vercel 서비스를 이용해 웹 서비스를 구축하기로 결정
 * Vercel에서 환경을 구축하는 과정에서, 이미 유출된 API 키를 삭제 후, 다시 수정 및 환경변수 수정
 * Vercel 환경에서 별도의 실행명령이 없으면 백엔드 서버를 실행하지 않음을 깨닫고, 로직을 수정함
 * VSC에서 별도로 서버를 실행하지 않아도, 링크만 접속하여 지금까지 구현한 서비스를 작동시켜볼 수 있는 웹 배포 성공
